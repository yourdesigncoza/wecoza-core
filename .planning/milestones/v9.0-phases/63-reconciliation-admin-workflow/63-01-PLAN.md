Extend the existing `assets/js/classes/agent-invoice.js` module (inside the existing jQuery IIFE) with reconciliation table functionality. Add three new functions and wire them into the existing init flow.

**1. Add `loadReconciliationTable()` call in the DOM ready block:**

In the existing `$(document).ready()`, after the `calculateInvoice()` call, add:
```js
if (config.isAdmin) {
    loadReconciliationTable();
```

**2. Create `loadReconciliationTable()` function:**

- Show `#reconciliation-loading`, hide `#reconciliation-table`, `#reconciliation-empty`, clear `#reconciliation-alert`
- POST to `wecoza_invoice_list` with {action, _ajax_nonce: ordersNonce, class_id: classId, agent_id: classAgent}
- On success with invoices array:
  - If empty: show `#reconciliation-empty`, hide loading
  - If non-empty: call `renderReconciliationRows(invoices)`, show `#reconciliation-table`, hide loading
- On error: show error in `#reconciliation-alert` using same alert pattern as `showInvoiceAlert()`

**3. Create `renderReconciliationRows(invoices)` function:**

Build table rows from invoices array. For each invoice:

```js
const month = (inv.invoice_month || '').substring(0, 7);
const monthDisplay = month ? new Date(month + '-15').toLocaleDateString('en-ZA', { year: 'numeric', month: 'long' }) : '—';
const classHours = formatHours(inv.class_hours_total);
const claimed = formatHours(inv.agent_claimed_hours);
const payable = formatHours(inv.calculated_payable_hours);
const discrepancy = parseFloat(inv.discrepancy_hours) || 0;
const status = inv.status || 'draft';
```

Row CSS class logic (applied to `<tr>`):
- If `discrepancy > 0` (overclaim: claimed > payable): add class `table-danger` (red highlight)
- If `discrepancy === 0 && status !== 'draft'`: add class `table-success` (green highlight) — only for non-draft rows that have been submitted

Discrepancy cell content:
- If `discrepancy > 0`: show `<span class="text-danger fw-bold">+{discrepancy} hrs</span>`
- If `discrepancy < 0`: show `<span class="text-success">{discrepancy} hrs</span>`
- If `discrepancy === 0`: show `<span class="text-success"><i class="bi bi-check-circle"></i> 0</span>`
- For draft invoices (no claimed hours yet): show `—`

Status cell: Use same `statusBadgeClass()` helper. Render as Phoenix badge:
```js
'<span class="badge badge-phoenix ' + statusBadgeClass(status) + '">' + escapeHtml(capitalise(status)) + '</span>'
```

Actions cell:
- If `status === 'submitted'`: render TWO buttons:
  ```html
  <button class="btn btn-phoenix-success btn-sm me-1 btn-approve" data-invoice-id="{invoice_id}">
    <i class="bi bi-check-lg"></i> Approve
  </button>
  <button class="btn btn-phoenix-danger btn-sm btn-dispute" data-invoice-id="{invoice_id}">
    <i class="bi bi-x-lg"></i> Dispute
  </button>
  ```
- If `status === 'approved'`: show `<span class="text-success small"><i class="bi bi-check-circle-fill"></i> Approved</span>` (no buttons)
- If `status === 'disputed'`: show `<span class="text-danger small"><i class="bi bi-exclamation-triangle-fill"></i> Disputed</span>` (warning indicator)
- If `status === 'draft'`: show `<span class="text-muted small">Pending submission</span>`

Set the built HTML into `#reconciliation-tbody`.

**4. Create `handleReviewAction(invoiceId, newStatus)` function:**

- Find the row's action cell (`button[data-invoice-id="X"]` closest `<tr>` find `td:last`)
- Replace action cell content with spinner
- POST to `wecoza_invoice_review` with {action: 'wecoza_invoice_review', _ajax_nonce: ordersNonce, invoice_id: invoiceId, status: newStatus}
- On success:
  - Update the status badge cell in the same row
  - Update the action cell:
    - approved: show green "Approved" text (no buttons)
    - disputed: show red "Disputed" warning text (no buttons)
  - No need to reload full table — just update the affected row in-place
- On error: restore buttons, show alert

**5. Add event bindings for approve/dispute buttons:**

In the existing `bindEvents()` function, add:
```js
$(document).on('click', '.btn-approve', function () {
    const invoiceId = parseInt($(this).data('invoice-id'), 10);
    if (invoiceId > 0) {
        handleReviewAction(invoiceId, 'approved');
    }
});
$(document).on('click', '.btn-dispute', function () {
    const invoiceId = parseInt($(this).data('invoice-id'), 10);
    if (invoiceId > 0) {
        handleReviewAction(invoiceId, 'disputed');
    }
});
```

**6. After a successful submitInvoice(), refresh the reconciliation table:**

In the existing `submitInvoice()` success handler (where it calls `showSubmittedState(invoice)`), add:
```js
if (config.isAdmin) {
    loadReconciliationTable();
}
```

This ensures the table reflects the newly submitted invoice without page reload.

- Reuse existing helper functions: `formatHours()`, `escapeHtml()`, `capitalise()`, `statusBadgeClass()`
- Use delegated event handlers via `$(document).on()` (same pattern as existing bindings)
- Error handling: try/catch or AJAX error callback, show user-friendly message, never leave buttons in disabled state
