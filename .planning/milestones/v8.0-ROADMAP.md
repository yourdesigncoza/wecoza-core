# Roadmap: v8.0 Page Tracking & Report Extraction

## Overview

Extends existing attendance capture and progression infrastructure with workbook page-number tracking, then builds a per-class report extraction feature with all fields Mario specified. Three phases: capture the data, surface it in progression views, then extract it as CSV reports.

## Phases

**Phase Numbering:** Continues from v7.0 (phases 53-55). Starts at 56.

- [ ] **Phase 56: Page Number Capture** - Add page number field to attendance modal and persist per learner per session
- [ ] **Phase 57: Page Progression Display** - Calculate and display page-based progression alongside hours-based progression
- [ ] **Phase 58: Report Extraction** - Per-class CSV report with header, learner details, hours, progression, and demographics

## Phase Details

### Phase 56: Page Number Capture
**Goal**: Agents can record the current workbook page number for each learner during attendance capture
**Depends on**: Nothing (first phase of v8.0; builds on v6.0/v7.0 attendance infrastructure)
**Requirements**: PAGE-01, PAGE-02
**Success Criteria** (what must be TRUE):
  1. Agent sees a "Page Number" input field per learner row in the attendance capture modal
  2. Page number is saved when agent submits attendance and persists in the database per learner per session
  3. Previously captured sessions show the recorded page number in the view modal
  4. Page number field is optional (attendance can still be submitted without it)
**Plans**: TBD

Plans:
- [ ] 56-01: Schema + backend (add page_number column, update AttendanceService/Repository, AJAX handler)
- [ ] 56-02: Frontend (add page number input to capture modal, display in view modal)

### Phase 57: Page Progression Display
**Goal**: Admins can see page-based progression metrics alongside hours-based progression for each learner
**Depends on**: Phase 56
**Requirements**: PAGE-03, PAGE-04
**Success Criteria** (what must be TRUE):
  1. Actual page progression percentage (current page / total pages in module) is calculated and displayed per learner
  2. Page progression column appears on the progression admin panel next to hours-based progression
  3. Learners without page data show a dash or "N/A" instead of broken display
**Plans**: TBD

Plans:
- [ ] 57-01: Progression calculation + admin panel integration (ProgressionService page % calc, admin panel column, SQL/CTE updates)

### Phase 58: Report Extraction
**Goal**: Admins can generate and download a per-class CSV report with complete learner details, hours, progression, and demographics
**Depends on**: Phase 57 (needs page progression data for RPT-05)
**Requirements**: RPT-01, RPT-02, RPT-03, RPT-04, RPT-05, RPT-06, RPT-07
**Success Criteria** (what must be TRUE):
  1. Admin can select a class and generate a report from a dedicated report extraction page/shortcode
  2. Report CSV header section contains: Client Name, Site Name, Class Type & Subject, Month, Class Days, Class Times, Facilitator
  3. Report CSV per-learner rows contain: Surname & Initials, Current Level/Module, Start Date, Race, Gender
  4. Report CSV hours columns contain: Current Month Trained, Current Month Present, Total Trained, Total Present
  5. Report CSV progression columns contain: Hours-based %, Actual page progression %
**Plans**: TBD

Plans:
- [ ] 58-01: Report service + data aggregation (ReportService, SQL queries joining class/learner/attendance/progression data)
- [ ] 58-02: Shortcode + controller + CSV download (shortcode UI, class selector, CSV generation endpoint)
