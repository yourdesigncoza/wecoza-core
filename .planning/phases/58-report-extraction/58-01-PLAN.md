Create `ReportRepository` in namespace `WeCoza\Classes\Repositories`, extending `BaseRepository`.

**Method 1: `getClassHeader(int $classId): ?array`**
Query that returns a single row with all class header fields:
- `client_name` (from clients table via classes.client_id)
- `site_name` (from sites table via classes.site_id)
- `class_type_name` (from class_types via classes.class_type)
- `class_subject` / `subject_name` (from class_type_subjects via classes.class_subject matching class_type_subject_id)
- `class_agent_name` (from agents table via classes.class_agent -- CONCAT first_name, surname)
- `schedule_data` (raw JSONB from classes -- caller will format days/times)
- `class_code`

SQL JOINs: classes -> clients, classes -> sites, classes -> class_types, classes -> class_type_subjects (on classes.class_subject = cts.class_type_subject_id), classes -> agents (on classes.class_agent = agents.agent_id)

Use try/catch with error_log pattern like other repositories.

**Method 2: `getClassLearnerReport(int $classId, int $year, int $month): array`**
Query returning one row per learner enrolled in the class with:
- `surname`, `first_name` (from learners)
- `race`, `gender` (from learners -- demographics)
- `subject_name` (current level/module from class_type_subjects)
- `start_date` (from learner_lp_tracking)
- `hours_trained` (total from learner_lp_tracking)
- `hours_present` (total from learner_lp_tracking)
- `subject_duration` (from class_type_subjects -- for hours % calc)
- `total_pages` (from class_type_subjects)
- `last_page_number` (from CTE — see below)
- Monthly hours: from CTE — see below

**IMPORTANT — Use CTEs instead of correlated subqueries (performance):**

CTE 1 — Monthly hours (aggregated once, not per-row):
```sql
WITH monthly_hours AS (
    SELECT learner_id, tracking_id,
           COALESCE(SUM(hours_trained), 0) AS month_hours_trained,
           COALESCE(SUM(hours_present), 0) AS month_hours_present
    FROM learner_hours_log
    WHERE class_id = :class_id
      AND log_date BETWEEN :month_start AND :month_end
    GROUP BY learner_id, tracking_id
)
```

CTE 2 — Page numbers (aggregated once, not per-row):
```sql
, page_numbers AS (
    SELECT (elem->>'learner_id')::int AS learner_id,
           cas.class_id,
           MAX((elem->>'page_number')::int) AS last_page_number
    FROM class_attendance_sessions cas,
         jsonb_array_elements(cas.learner_data) AS elem
    WHERE cas.class_id = :class_id
      AND elem->>'page_number' IS NOT NULL
      AND elem->>'page_number' ~ '^[0-9]+$'
    GROUP BY (elem->>'learner_id')::int, cas.class_id
)
```
Note: The `~ '^[0-9]+$'` regex guard prevents ::int cast failures on non-numeric page_number values.

Main query then LEFT JOINs both CTEs:
```sql
SELECT ... FROM learner_lp_tracking lpt
JOIN learners l ON lpt.learner_id = l.id
JOIN class_type_subjects cts ON lpt.class_type_subject_id = cts.class_type_subject_id
LEFT JOIN monthly_hours mh ON mh.learner_id = lpt.learner_id AND mh.tracking_id = lpt.tracking_id
LEFT JOIN page_numbers pn ON pn.learner_id = lpt.learner_id AND pn.class_id = lpt.class_id
WHERE lpt.class_id = :class_id
```

Use COALESCE for nullable fields: `COALESCE(l.race, '')`, `COALESCE(l.gender, '')`, `COALESCE(mh.month_hours_trained, 0)`, etc.

Filter: WHERE lpt.class_id = :class_id
Order: l.surname, l.first_name

Use PDO prepared statements with named params. Follow the same error handling pattern as LearnerProgressionRepository.

  <action>
Create `ReportService` in namespace `WeCoza\Classes\Services`.

Constructor: accepts optional ReportRepository (default creates new instance).

**Method 1: `generateClassReport(int $classId, int $year, int $month): array`**
Returns structured report data:
```php
[
    'header' => [...],      // from getClassHeader, enriched
    'learners' => [...],    // from getClassLearnerReport, enriched
    'meta' => ['generated_at' => ..., 'month_label' => ..., 'class_id' => ...]
]
```

Enrichment for header:
- Parse `schedule_data` JSONB to extract class_days (comma-separated day names like "Mon, Wed, Fri") and class_times (formatted like "08:00-12:00")
- Facilitator = agent name from header query
- Month label = e.g. "March 2026" from $year/$month

Enrichment for learners:
- `initials` = first letter of first_name with dot (e.g. "J.")
- `hours_progress_pct` = `($subject_duration > 0) ? min(round(($hours_present / $subject_duration) * 100, 1), 100.0) : null` — MUST guard against division by zero
- `page_progress_pct` = `($total_pages > 0) ? min(round(($last_page_number / $total_pages) * 100, 1), 100.0) : null` — MUST guard against division by zero
- Both show null/dash when data missing (not 0)

**Method 2: `formatCsvRows(array $reportData): array`**
Returns array of arrays ready for fputcsv(). Structure:
- Row 1 (header info): Client, [client_name]
- Row 2: Site, [site_name]
- Row 3: Class Type & Subject, [class_type_name - subject_name]
- Row 4: Month, [month_label]
- Row 5: Class Days, [class_days]
- Row 6: Class Times, [class_times]
- Row 7: Facilitator, [facilitator]
- Row 8: empty row
- Row 9: Column headers: Surname, Initials, Current Level/Module, Start Date, Race, Gender, Month Trained, Month Present, Total Trained, Total Present, Hours Progress %, Page Progress %
- Rows 10+: Learner data rows matching column headers
- If no learners: single row under headers with "No learners found" in first column

**IMPORTANT:** Pad all metadata rows (1-8) with empty strings so every row has exactly 12 columns (matching the data table). This prevents Excel from complaining about inconsistent row lengths.

Handle empty/null fields with empty string fallback. Format dates as d/m/Y. Format percentages with 1 decimal + "%" suffix.
