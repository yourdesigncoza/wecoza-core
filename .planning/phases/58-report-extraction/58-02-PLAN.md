  - Download CSV button (`btn btn-phoenix-success`, initially hidden/disabled)

**3. AJAX handlers: `src/Classes/Ajax/ReportAjaxHandlers.php`**
Namespace: `WeCoza\Classes\Ajax`.

Handler 1: `handle_generate_class_report()` (wp_ajax action: `generate_class_report`)
- Verify nonce with `check_ajax_referer('report_extraction_nonce', 'nonce')`
- Get `class_id` (int), `year` (int), `month` (int) from POST
- Validate: class_id required, year/month default to current if missing
- Call `ReportService::generateClassReport($classId, $year, $month)`
- Return JSON: `wp_send_json_success($reportData)`

Handler 2: `handle_download_class_report_csv()` (wp_ajax action: `download_class_report_csv`)
- Verify nonce with `check_ajax_referer('report_extraction_nonce', 'nonce')` -- NOTE: this uses GET params since it triggers a download
- Get `class_id`, `year`, `month` from GET
- Call ReportService to generate then formatCsvRows
- Stream CSV using the same pattern as handle_export_regulatory_csv: Content-Type, Content-Disposition, UTF-8 BOM, fputcsv loop
- Filename format: `class-report-{class_code}-{year}-{month}.csv`
- `exit()` after streaming (not wp_die)

Register function: `register_report_ajax_handlers()` called via `add_action('init', ...)`.

**4. JS file: `assets/js/classes/report-extraction.js`**
jQuery-based (matches project convention). On document ready:

- Populate class dropdown from `reportExtractionAjax.classes`
- Set month picker default to current month (YYYY-MM format)
- Generate button click: POST to admin-ajax.php with action=generate_class_report, class_id, year, month, nonce. On success, render preview HTML into `#report-preview` and show/enable Download CSV button.
- Download CSV button click: Construct URL with GET params (action=download_class_report_csv, class_id, year, month, nonce) and trigger download via `window.location.href` (same pattern used by regulatory export).

Preview rendering: Build a header info section (dl/dt/dd pairs) and a Bootstrap table for learner rows. Use Phoenix table classes (`table table-sm table-hover`). Show dash for null values. Format percentages with 1 decimal.

CSS: NO new CSS file. If any custom styles needed, append to `/opt/lampp/htdocs/wecoza/wp-content/themes/wecoza_3_child_theme/includes/css/ydcoza-styles.css`. Prefer Phoenix/Bootstrap classes.
