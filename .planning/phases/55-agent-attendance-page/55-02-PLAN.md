**Part A: Add redirect hooks to AgentAccessController constructor**

Add three new hooks to the existing constructor (after the two `init` hooks from Plan 55-01):

```php
add_filter('login_redirect', [$this, 'redirectAgentOnLogin'], 9, 3);
add_action('admin_init', [$this, 'blockAgentAdminAccess']);
add_action('template_redirect', [$this, 'enforceAgentPageCage']);
```

**Part B: Implement three new public methods**

1. `redirectAgentOnLogin(string $redirectTo, string $request, $user): string`
   - If `$user` is not a `\WP_User` instance, return `$redirectTo` unchanged
   - If `in_array('wp_agent', $user->roles, true)`, return `home_url('/agent-attendance/')`
   - Otherwise return `$redirectTo` unchanged
   - Priority 9 ensures this fires BEFORE the theme's priority-10 `ydcoza_force_login_redirect_to_home` filter

2. `blockAgentAdminAccess(): void`
   - FIRST: `if (wp_doing_ajax()) { return; }` — CRITICAL: never block admin-ajax.php (attendance capture uses it)
   - Get `$user = wp_get_current_user()`
   - If `in_array('wp_agent', $user->roles, true)`, call `wp_redirect(home_url('/agent-attendance/'))` then `exit`
   - Otherwise return (no-op for admins/other roles)

3. `enforceAgentPageCage(): void`
   - If `!is_user_logged_in()`, return
   - Get `$user = wp_get_current_user()`
   - If NOT `in_array('wp_agent', $user->roles, true)`, return (non-agents unaffected)
   - Check allowlist — allow these pages without redirect:
     a. `is_page('agent-attendance')` — the attendance landing page
     b. `get_page_by_path('app/display-single-class')` exists AND `is_page()` matches its ID — single class view for capture
   - If current page is NOT in allowlist: `wp_redirect(home_url('/agent-attendance/'))` then `exit`
   - If attendance page lookup returns null (page doesn't exist), log error via `wecoza_log()` and do NOT redirect (prevents infinite loop per research pitfall 5)
