<task type="auto">
  <name>Task 2: Create agent-invoice.js with AJAX interactions</name>
  <files>assets/js/classes/agent-invoice.js</files>
  <action>
Create `assets/js/classes/agent-invoice.js` — a self-contained module using the same jQuery AJAX pattern as attendance-capture.js.

**Initialization (on DOMContentLoaded):**
- Read from `WeCozaSingleClass`: classId, classAgent (agent_id), ajaxUrl, ordersNonce, isAdmin
- If classAgent is 0 or falsy, return early (no agent assigned)
- Call `loadOrder()` to populate rate card (if admin)
- Call `calculateInvoice()` with current month to populate summary on load

**loadOrder() function:**
- POST to `wecoza_order_get` with {action, _ajax_nonce: ordersNonce, class_id: classId, agent_id: classAgent}
- On success: store order.order_id in `#agent-order-id`, set `#agent-rate-type` to order.rate_type, set `#agent-rate-amount` to order.rate_amount, enable inputs
- On 404 (no order): show "No order found" message in rate card, keep inputs disabled

**saveRate() function (bound to #btn-save-agent-rate click):**
- Read order_id from `#agent-order-id`, rate_type from `#agent-rate-type`, rate_amount from `#agent-rate-amount`
- Validate: order_id > 0, rate_type not empty, rate_amount >= 0
- POST to `wecoza_order_save` with {action, _ajax_nonce: ordersNonce, order_id, rate_type, rate_amount}
- On success: flash green "Saved" in `#agent-rate-status`, fade after 2s
- On error: show red error message

**calculateInvoice() function:**
- Read month from `#invoice-month-picker` (YYYY-MM format)
- Show `#invoice-loading`, hide `#invoice-summary`, `#invoice-claim-form`, `#invoice-submitted`
- POST to `wecoza_invoice_calculate` with {action, _ajax_nonce: ordersNonce, class_id: classId, agent_id: classAgent, invoice_month: month}
- On success: populate #inv-class-hours, #inv-absent-days, #inv-absent-hours, #inv-payable-hours
- Show `#invoice-summary`
- Then call `loadInvoiceStatus(month)` to determine if claim form or submitted state should show

**loadInvoiceStatus(month) function:**
- If admin: POST to `wecoza_invoice_list` with {action, _ajax_nonce: ordersNonce, class_id: classId, agent_id: classAgent}
- Find invoice matching the selected month (compare invoice_month YYYY-MM-01 prefix)
- If admin but no matching invoice or invoice status is 'draft': show `#invoice-claim-form` (agents fill this)
- If wp_agent (not admin): POST to `wecoza_invoice_calculate` already done; check if an invoice exists for this month. Since wecoza_invoice_list requires manage_options, the agent cannot call it. Instead, attempt wecoza_invoice_submit with a dry-run approach -- NO, that would write. Better approach: add the invoice status to the calculateInvoice response.

**IMPORTANT SIMPLIFICATION:** Since the calculate endpoint is pure-read and available to capture_attendance, modify the approach: after calculateInvoice succeeds, check if the response includes an existing invoice status. Looking at the Phase 60 code, handleCalculate only returns summary data, not invoice status. Rather than modifying the backend, use this approach:
- For agents: always show the claim form. If they try to submit and the invoice is already submitted, the backend will handle it (submitInvoice calls findOrCreateDraft which returns existing row).
- For admins: call wecoza_invoice_list to get the invoice status and show the appropriate state.
- To detect "already submitted" for agents: after a successful submit, store the month in a local Set. On page reload, the calculate call won't know, but the submit endpoint will return the current status. This is acceptable UX.

**REVISED loadInvoiceStatus(month):**
- If isAdmin: call wecoza_invoice_list, find matching month invoice, show submitted state if status != 'draft', else show claim form
- If NOT admin (agent): always show claim form. On submit success, swap to submitted state.
- Pre-fill #inv-claimed-hours with the calculated_payable_hours value as a convenience default

**submitInvoice() function (bound to #btn-submit-invoice click):**
- Read claimed_hours from `#inv-claimed-hours`, notes from `#inv-claim-notes`, month from `#invoice-month-picker`
- Validate: claimed_hours >= 0
- Disable button, show spinner
- POST to `wecoza_invoice_submit` with {action, _ajax_nonce: ordersNonce, class_id: classId, agent_id: classAgent, invoice_month: month, claimed_hours, notes}
- On success: hide `#invoice-claim-form`, show `#invoice-submitted` with:
  - Status badge (submitted = badge-phoenix-info)
  - "Claimed Hours: X" text
  - If discrepancy_hours from response > 0: warning badge "Overclaim: X hrs"
  - Notes text if provided
- On error: re-enable button, show error

**Month picker change handler:**
- Bind `#invoice-month-picker` change to `calculateInvoice()`

**Event bindings:**
- `$(document).on('click', '#btn-save-agent-rate', saveRate)`
- `$(document).on('click', '#btn-submit-invoice', submitInvoice)`
- `$(document).on('change', '#invoice-month-picker', function() { calculateInvoice(); })`

Use consistent error handling pattern: try/catch on AJAX, show user-friendly message on failure, always re-enable buttons on error.
