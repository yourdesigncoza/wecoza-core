Create three SQL migration files:

**File 1: schema/001_create_agent_orders.sql**

CREATE TABLE public.agent_orders with columns:
- order_id SERIAL PRIMARY KEY
- class_id INTEGER NOT NULL REFERENCES public.classes(class_id)
- agent_id INTEGER NOT NULL REFERENCES public.agents(agent_id)
- rate_type VARCHAR(20) NOT NULL DEFAULT 'hourly' CHECK (rate_type IN ('hourly', 'daily'))
- rate_amount NUMERIC(10,2) NOT NULL DEFAULT 0.00 CHECK (rate_amount >= 0)
- start_date DATE NOT NULL DEFAULT CURRENT_DATE
- end_date DATE (nullable — NULL means "current/ongoing")
- notes TEXT (nullable)
- created_at TIMESTAMP DEFAULT NOW()
- updated_at TIMESTAMP DEFAULT NOW()
- created_by INTEGER (nullable — wp user id)

Add UNIQUE constraint: UNIQUE(class_id, agent_id, start_date) — this is the key constraint enabling rate changes (same class+agent, different start_date = new rate period).

Add index: CREATE INDEX idx_agent_orders_class_agent ON public.agent_orders(class_id, agent_id).

Add trigger: Reuse existing update_updated_at_column() function for updated_at auto-update.

Add table and column comments matching the commenting style in the existing schema dump.

Owner: SET TO "John" (matching all existing tables).

**File 2: schema/002_create_agent_monthly_invoices.sql**

CREATE TABLE public.agent_monthly_invoices with columns:
- invoice_id SERIAL PRIMARY KEY
- order_id INTEGER NOT NULL REFERENCES public.agent_orders(order_id) ON DELETE RESTRICT
- class_id INTEGER NOT NULL (denormalized — per STATE.md decision)
- agent_id INTEGER NOT NULL (denormalized — per STATE.md decision)
- invoice_month DATE NOT NULL (first day of month, e.g. '2026-03-01')
- class_hours_total NUMERIC(8,2) NOT NULL DEFAULT 0.00 (system-calculated total scheduled hours)
- all_absent_days INTEGER NOT NULL DEFAULT 0 (count of sessions where all learners had 0 hours)
- all_absent_hours NUMERIC(8,2) NOT NULL DEFAULT 0.00 (hours from all-absent sessions)
- calculated_payable_hours NUMERIC(8,2) NOT NULL DEFAULT 0.00 (class_hours_total minus all_absent_hours)
- agent_claimed_hours NUMERIC(8,2) (nullable — agent fills this in on submission)
- agent_notes TEXT (nullable)
- discrepancy_hours NUMERIC(8,2) (nullable — agent_claimed_hours minus calculated_payable_hours; positive = overclaim)
- status VARCHAR(20) NOT NULL DEFAULT 'draft' CHECK (status IN ('draft', 'submitted', 'approved', 'disputed'))
- submitted_at TIMESTAMP (nullable)
- reviewed_at TIMESTAMP (nullable)
- reviewed_by INTEGER (nullable — wp user id of admin who approved/disputed)
- review_notes TEXT (nullable)
- created_at TIMESTAMP DEFAULT NOW()
- updated_at TIMESTAMP DEFAULT NOW()

Add UNIQUE constraint: UNIQUE(order_id, invoice_month) — one invoice per order per month.
Add index: CREATE INDEX idx_agent_invoices_class_month ON public.agent_monthly_invoices(class_id, invoice_month).
Add index: CREATE INDEX idx_agent_invoices_agent_month ON public.agent_monthly_invoices(agent_id, invoice_month).
Add index: CREATE INDEX idx_agent_invoices_status ON public.agent_monthly_invoices(status).

Add trigger: Reuse existing update_updated_at_column() function.

Add FK constraints to classes and agents tables for the denormalized columns (these ensure data integrity even though denormalized).

Add table and column comments. Owner: "John".

**File 3: schema/003_seed_agent_orders_from_classes.sql**

Write a migration that:
1. Inserts into agent_orders for every class WHERE class_status = 'active' AND class_agent IS NOT NULL
2. Uses class_id, class_agent as agent_id, rate_type='hourly', rate_amount=0.00 (admin sets real rates later — per STATE.md blocker note), start_date = COALESCE(original_start_date, CURRENT_DATE)
3. Uses ON CONFLICT DO NOTHING to be idempotent (safe to run multiple times)
4. Wraps in a transaction (BEGIN/COMMIT)
5. Adds a summary SELECT count(*) at the end so the user sees how many rows were inserted

Each file should start with a comment block: filename, purpose, date, and a note that it must be run in order (001 before 002 before 003).
