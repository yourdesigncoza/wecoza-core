  <action>
Create two repository classes extending BaseRepository.

**AgentOrderRepository** (`src/Agents/Repositories/AgentOrderRepository.php`):
- Namespace: `WeCoza\Agents\Repositories`
- Extends `WeCoza\Core\Abstract\BaseRepository`
- `$table = 'agent_orders'`, `$primaryKey = 'order_id'`
- Column whitelisting: insert allows [class_id, agent_id, rate_type, rate_amount, start_date, end_date, notes, created_at, updated_at, created_by]; update allows [rate_amount, rate_type, end_date, notes, updated_at]; filter allows [order_id, class_id, agent_id, status, start_date]; order allows [order_id, class_id, agent_id, start_date, created_at]
- `findActiveOrderForClass(int $classId, int $agentId): ?array` — raw SQL: SELECT * WHERE class_id = :class_id AND agent_id = :agent_id AND (end_date IS NULL OR end_date >= CURRENT_DATE) ORDER BY start_date DESC LIMIT 1
- `ensureOrderExists(int $classId, int $agentId, string $startDate, int $createdBy): ?int` — raw SQL INSERT with ON CONFLICT (class_id, agent_id, start_date) DO NOTHING, then SELECT order_id for the class+agent (ORDER BY start_date DESC LIMIT 1). Use `$this->db->query()` for raw SQL, NOT BaseRepository::insert(). Returns order_id.
- `findOrdersForClass(int $classId): array` — findBy(['class_id' => $classId]) ordered by start_date DESC. For the admin rate card view.

**AgentInvoiceRepository** (`src/Agents/Repositories/AgentInvoiceRepository.php`):
- Namespace: `WeCoza\Agents\Repositories`
- Extends `WeCoza\Core\Abstract\BaseRepository`
- `$table = 'agent_monthly_invoices'`, `$primaryKey = 'invoice_id'`
- Column whitelisting: insert allows [order_id, class_id, agent_id, invoice_month, class_hours_total, all_absent_days, all_absent_hours, calculated_payable_hours, agent_claimed_hours, discrepancy_hours, notes, status, reviewed_by, reviewed_at, created_at, updated_at]; update allows [class_hours_total, all_absent_days, all_absent_hours, calculated_payable_hours, agent_claimed_hours, discrepancy_hours, notes, status, reviewed_by, reviewed_at, updated_at]; filter allows [invoice_id, order_id, class_id, agent_id, invoice_month, status]; order allows [invoice_id, invoice_month, status, created_at]
- `findOrCreateDraft(int $orderId, int $classId, int $agentId, string $invoiceMonth): array` — raw SQL INSERT with ON CONFLICT (order_id, invoice_month) DO NOTHING, then SELECT the row. Returns full invoice row array.
- `getSessionsForMonth(int $classId, string $invoiceMonth): array` — raw SQL: SELECT session_id, session_date, scheduled_hours, status, learner_data FROM class_attendance_sessions WHERE class_id = :class_id AND session_date >= :month_start AND session_date < :month_end ORDER BY session_date ASC. Calculate month_start as first day, month_end as first day of next month. Use `$this->db->query()`.
- `findInvoicesForClassAgent(int $classId, int $agentId, int $limit = 12): array` — findBy(['class_id' => $classId, 'agent_id' => $agentId]) ordered by invoice_month DESC. For reconciliation list view.

Follow existing patterns: `declare(strict_types=1)`, ABSPATH guard, proper PHPDoc on all methods. Use `$this->db->query($sql, $params)` for all raw SQL — this returns a PDOStatement. Fetch with `->fetch(\PDO::FETCH_ASSOC)` or `->fetchAll(\PDO::FETCH_ASSOC)`.

  <action>
Create two service classes following the AgentService pattern (constructor creates repository instances).

**AgentOrderService** (`src/Agents/Services/AgentOrderService.php`):
- Namespace: `WeCoza\Agents\Services`
- Private property: `AgentOrderRepository $orderRepo`
- Constructor: `$this->orderRepo = new AgentOrderRepository()`
- `ensureOrderForClass(int $classId, int $agentId, ?string $originalStartDate = null): ?int` — if $originalStartDate is null, use `date('Y-m-d')` as start_date. Call `$this->orderRepo->ensureOrderExists($classId, $agentId, $startDate, get_current_user_id())`. Returns order_id or null.
- `getActiveOrder(int $classId, int $agentId): ?array` — delegates to `$this->orderRepo->findActiveOrderForClass($classId, $agentId)`
- `saveOrderRate(int $orderId, string $rateType, float $rateAmount): bool` — validates rateType in ['hourly', 'daily'], calls `$this->orderRepo->update($orderId, ['rate_type' => $rateType, 'rate_amount' => $rateAmount, 'updated_at' => date('Y-m-d H:i:s')])`. Returns bool.

**AgentInvoiceService** (`src/Agents/Services/AgentInvoiceService.php`):
- Namespace: `WeCoza\Agents\Services`
- Private properties: `AgentInvoiceRepository $invoiceRepo`, `AgentOrderRepository $orderRepo`
- Constructor creates both repositories

- `calculateMonthSummary(int $classId, int $agentId, string $invoiceMonth): array` — Core calculation method:
  1. Normalize $invoiceMonth to 'Y-m-01' format
  2. Call `$this->invoiceRepo->getSessionsForMonth($classId, $invoiceMonth)`
  3. Loop sessions: sum `scheduled_hours` for `class_hours_total`; detect all-absent sessions using private `isAllAbsentSession()` method; count `all_absent_days` and sum their `scheduled_hours` as `all_absent_hours`
  4. `calculated_payable_hours = class_hours_total - all_absent_hours`
  5. Return associative array: `['class_hours_total' => float, 'all_absent_days' => int, 'all_absent_hours' => float, 'calculated_payable_hours' => float, 'sessions_captured' => int, 'sessions' => array]`

- `private isAllAbsentSession(array $session): bool` — Logic branching on status:
  - `'agent_absent'` → return true
  - `'captured'` → parse learner_data (json_decode if string), return true only if ALL entries have hours_present == 0. If learner_data is empty/null, return false.
  - Any other status ('pending', 'client_cancelled') → return false
  - Handle learner_data being string (json_decode) or already array

- `submitInvoice(int $classId, int $agentId, string $invoiceMonth, float $claimedHours, ?string $notes = null): array` —
  1. Find active order via `$this->orderRepo->findActiveOrderForClass($classId, $agentId)`. If none, throw `\RuntimeException('No active order found')`
  2. Calculate month summary via `$this->calculateMonthSummary()`
  3. `$discrepancy = $claimedHours - $summary['calculated_payable_hours']`
  4. Find-or-create draft via `$this->invoiceRepo->findOrCreateDraft($order['order_id'], $classId, $agentId, $invoiceMonth)`
  5. Update invoice row: set class_hours_total, all_absent_days, all_absent_hours, calculated_payable_hours, agent_claimed_hours, discrepancy_hours, notes, status='submitted', updated_at
  6. Return updated invoice row

- `reviewInvoice(int $invoiceId, string $newStatus, int $reviewedBy): bool` — validates $newStatus in ['approved', 'disputed']. Updates invoice: status, reviewed_by, reviewed_at=now(), updated_at.

- `getInvoicesForClassAgent(int $classId, int $agentId, int $limit = 12): array` — delegates to repo

Follow existing patterns: `declare(strict_types=1)`, ABSPATH guard, proper PHPDoc.
