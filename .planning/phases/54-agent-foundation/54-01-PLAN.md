Add a new `plugins_loaded` hook at priority 6 (after core init at priority 5) that registers `capture_attendance` capability on both `wp_agent` and `administrator` roles.

**Location:** Add AFTER the existing `plugins_loaded` block (after the closing `);` around line 188). Do NOT add inside the existing priority-5 block.

**Code to add:**

```php
/*
|--------------------------------------------------------------------------
| Register attendance capture capabilities
|--------------------------------------------------------------------------
|
| Uses plugins_loaded (not just activation hook) so capabilities survive
| plugin updates. add_cap() is idempotent — safe to call on every load.
|
*/

add_action(
    "plugins_loaded",
    function () {
        // Add capture_attendance to wp_agent role
        $agentRole = get_role('wp_agent');
        if ($agentRole) {
            $agentRole->add_cap('capture_attendance');
        }

        // Add capture_attendance to administrator role
        $adminRole = get_role('administrator');
        if ($adminRole) {
            $adminRole->add_cap('capture_attendance');
        }
    },
    6,
);
```

**Why no `has_cap()` guard:** Follow existing codebase convention (lines 761-771 call `add_cap()` directly without guards). `add_cap()` only writes to DB if value changes — negligible cost.

**Do NOT:**
- Modify the existing activation hook role registration (lines 750-756) — keep `wp_agent` creation as-is
- Add `capture_attendance` to the activation-hook admin block (lines 760-772) — the `plugins_loaded` hook covers both activation and update scenarios
- Remove any existing capabilities from either role
