$learnerData[] = [

**B) getSessionDetail method (lines ~499-527):**
In the `learner_data` JSONB fallback branch where it builds `$learners[]` from decoded JSONB, add `page_number` to the output:

```php
$learners[] = [
    'learner_id'    => $lid,
    'learner_name'  => $nameMap[$lid] ?? 'Unknown',
    'hours_trained' => number_format($scheduledHours, 2, '.', ''),
    'hours_present' => number_format($hoursPresent, 2, '.', ''),
    'hours_absent'  => number_format(max(0, $scheduledHours - $hoursPresent), 2, '.', ''),
    'page_number'   => (int) ($ld['page_number'] ?? 0),
];
```

Also check the `getSessionsWithLearnerHours` SQL path (repository method). That path queries `learner_hours_log` table which does NOT have page_number. For sessions loaded via that path, page_number should be extracted from the session's `learner_data` JSONB as a supplemental source. Add a helper approach:

After the `$learners = $this->repository->getSessionsWithLearnerHours($sessionId);` call (line ~497), if `$learners` is NOT empty, supplement each entry with page_number from the session's `learner_data` JSONB:

```php
if (!empty($learners) && !empty($session['learner_data'])) {
    $learnerDataRaw = is_string($session['learner_data'])
        ? json_decode($session['learner_data'], true)
        : $session['learner_data'];
    $pageMap = [];
    if (is_array($learnerDataRaw)) {
        foreach ($learnerDataRaw as $ld) {
            $pageMap[(int) ($ld['learner_id'] ?? 0)] = (int) ($ld['page_number'] ?? 0);
        }
    }
    foreach ($learners as &$l) {
        $l['page_number'] = $pageMap[(int) $l['learner_id']] ?? 0;
    }
    unset($l);
}
```

This ensures page_number is always in the detail response regardless of which data path is used.
